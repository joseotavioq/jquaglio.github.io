Any CPU, x86, x64, Itanium ou Mixed Platforms

Mar 18, 2014

  • .NET
  • Visual Studio 2010

O que são: Any CPU, x86, x64 e Itanium

São configurações de compilação que indicam a plataforma onde o assembly será executado.

Qual a diferença entre eles?

Any CPU - O assembly será compilado em uma forma independente, ou seja, tanto faz qual versão do .NET Framework (x86 ou x64) executará o assembly, ele será executado de acordo com a versão instalada.

x86 - O assembly será executado como x86, independente se o sistema operacional é x86 ou x64. Caso o assembly seja executado em uma arquitetura x64, o assembly será carregado em modo WOW64.

x64 - O assembly será executado somente como x64, não sendo possível sua execução em uma arquitetura x86.

Itanium - O assembly será executado somente em uma plataforma que contenha o processador Itanium.

Mixed Platforms - Indica que na solution existem projetos configurados para vários tipos de plataformas.

## Qual a melhor a ser utilizada?

Depende do cenário!

Recomendo utilizar o Any CPU quando o desenvolvedor não tem conhecimento da plataforma do servidor de produção.

Utilize x86 em cenários que necessitam de interoperabilidade com códigos não gerenciados ou quando o servidor de produção é somente x86.

Como alterar essa configuração no Microsoft Visual Studio 2010 para os meus projetos no SLN (solution)?

Antes de realizar a alteração de plataforma faça uma análise de impacto para que você não tenha problemas ao alterar um arquitetura x86 para x64, por exemplo!

Dentro do Microsoft Visual Studio 2010, logo abaixo do menu na barra de ícones, localize as listas que informam a configuração do projeto (Ex.: Debug, Release) e as plataformas (Any CPU, x86, x64, Itanium), conforme exemplo da imagem abaixo:

Clique na listagem de plataformas e selecione a plataforma que quiser e compile a aplicação.

Caso as outras opções de plataformas não apareçam na listagem ou você queira saber quais configurações de plataformas estão para cada projeto da solution, clique no item “Configuration Manager…”.

Ao abrir o Configuration Manager, serão exibidos todos os projetos contidos na solution.

Clique sobre a célula da coluna “Platform” e o projeto que deseja alterar e, selecione “New…”.

Irá abrir uma janela para que você adicione a nova plataforma a partir de outra existente.

Clique em “OK”. Agora verifique se o projeto foi alterado para a configuração que foi criada anteriormente.

Se estiver tudo ok, o próximo passo é efetuar um build do projeto que foi alterado com a nova configuração de plataforma.
